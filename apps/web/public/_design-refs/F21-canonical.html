D-2104 P0 In progress

Refund > ₹10K stuck on PROCESSING — webhook timeout on payments-svc when partner-bank gateway exceeds 30s SLA.

Summary

Refunds > ₹10K from partner-bank channel are stuck on PROCESSING when the partner gateway exceeds the 30 s webhook SLA. The retry pipeline marks them as failed but does not surface the error to the merchant.

1.Initiate refund > ₹10K via partner channel
2.Wait for webhook timeout (≥30 s)
3.Refresh — status remains PROCESSING

Root cause · Sherlock
Sherlock i 94% confidence
Webhook handler in payments-svc::PartnerCallbackController sets a 30 s read-timeout on the inbound socket. When the partner-bank gateway holds the connection longer (observed P95 = 41 s), the request is aborted before the status update is committed — leaving the refund row in PROCESSING.
