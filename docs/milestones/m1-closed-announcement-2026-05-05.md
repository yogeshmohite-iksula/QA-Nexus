## Key deliverables

### Auth

- **BetterAuth magic-link** — 10-min TTL, cross-subdomain session cookie
  (`Domain=.qanexus.iksula.com`, CHIPS `Partitioned: true`), `nextCookies()`
  plugin wired per BetterAuth issue #4038 guidance (PR #44, #46)
- **ADR-007** — cookie-domain decision documented; wildcard-parent strategy
  chosen over exact-domain + per-fetch-credentials (PR #43)
- **F06 Sign In** — magic-link UX with 4 view states (initial / submitting /
  sent / error toast) — PR #42

### RBAC

- **4-role matrix** (Admin / Lead / QA Engineer / Stakeholder) guarding all
  M1 endpoints — UsersController, InvitationsController,
  ProjectMembersController, AdminLlmConfigController, KbModule
- **Cross-workspace isolation** — every user-scoped query workspace-filtered;
  cross-workspace resource access returns 404 (no leak)
- **Roles sweep** — 54-test `@M1-CLOSE-GATE` e2e suite (PR #45)

### Audit chain

- **HMAC-SHA256 chained `audit_log`** — every state-change writes
  `this_hash = HMAC(secret, prev_hash || canonical(payload))`
  synchronously before response; 8 audit integrity tests in sweep
- **`scripts/verify-audit-chain.ts`** — standalone verifier runnable
  post-deploy against the live Neon DB

### Users & Roles admin surface

- **F27 Users & Roles** (`/admin/users`) — full team roster with role pills,
  status badges, invite panel, optimistic updates — PR #41 + PR #48
- **F27m1 Invite User Modal** — workspace invitation flow wired to
  `POST /api/invitations`, 7-day expiry, email delivery via Gmail SMTP
- **F28 Settings & Audit** (`/admin/settings`) — audit log surface live
