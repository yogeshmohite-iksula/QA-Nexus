## §8 Day-28 Wed additional lessons

1. **F28 was a long visual-gate cycle (~15 drift items / 7 rounds), opposite of F23's one-shot.** Root cause: F28 is content-dense (8 tabs, a 7-column audit table) and the diff-probe AMBER band doesn't catch fine per-element drift (badge sizes, focus rings, bg distinctions, colored sub-cells) — only Yogesh's eye does. Lesson: for dense admin frames, **extract the canonical CSS for EVERY sub-component up front** (`.act.*`, `.tn-lock`, `.imm-banner`, `.audit-tbl`, etc.) and match verbatim in the first scaffold pass, rather than approximating and iterating.
2. **Unlayered global `:focus-visible` in globals.css (line 128) beats layered Tailwind utilities** regardless of specificity. To suppress the violet focus ring on a specific input, the Tailwind **important modifier** (`outline-none!`) is required — plain `focus-visible:outline-none` loses to the unlayered global. Applies to any future input that shouldn't show the default ring.
3. **extract-canned-data captures text nodes only — NOT input `value=` or `<option>` text.** For form-heavy frames (General/Branding here), the displayed field values must be pulled tool-assisted (grep/JS on the canonical HTML's `value=`/`<option>`) and added to canned-data with a traceability comment. Filed as an extract-canned-data v-next enhancement candidate.
4. **extract-canned-data auto-slugifies `F28` → `f28/` dir**, not `f28-settings-audit/`. Handled with `mv` + `rmdir` to the Option-A admin path. Hygiene fix candidate: slugifier should honor a `--out` or frame→dir map.
5. **Canonical > MAIN brief when they conflict.** MAIN's Step-4 brief specified "actor role pills (ADMIN/LEAD/…)" but the canonical v2.3 audit table has NO role pills — just color-coded actors (agents violet + ⓘ, system italic). Followed the canonical (Rule 15) + removed the pills. Yogesh's visual gate (Rule 13) is authoritative; the canonical HTML is the content source-of-truth.
