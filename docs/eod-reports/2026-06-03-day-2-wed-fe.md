## §7 Lessons learned (the day's surprises)

Captured in detail at `.claude/memory/feedback_skill_v2.2_first_use.md` (this EOD). Headlines:

1. **Skill v2.2 diff-probe RED was a false-positive for FE+1**. 40% sections-PASS reflected AdminShell Tailwind divergence from canonical BEM (Day-19 amendment Part 1 expected) + modal/drawer Phase-3 scope, not real port drift. Yogesh visual gate (Hard Rule 13) was authoritative per Day-19 amendment Part 3. **v2.2.1 candidate:** sections-PASS threshold for "RED" should account for known divergence categories (AdminShell shell vs canonical BEM, Phase-3 modal scope) — currently 40% triggers RED but should be GREEN/AMBER for first-pass non-shell content.

2. **Canonical `:root` token block CANNOT be stripped during CSS extraction.** I extracted canonical `<style>` block to `agents-page.css` and stripped `:root` thinking globals.css had the tokens. Globals only had color tokens — was missing `--r-lg` (12px section radius), `--rail-w` (240px), `--topbar-h` (56px), `--tap` (44px). All `border-radius: var(--r-lg)` rules silently fell back to 0 → flat-corner sections. **Fix:** restore canonical `:root` scoped to `.main`. **For F27 + modals:** keep the scoped block, don't strip.

3. **Canonical mobile `grid-column: 1/-1` overrides REQUIRE explicit reset in mobile responsive layouts.** Canonical `.dec-row .conf { grid-column: 1/-1 }` is more specific than my row template and survives even with `!important` on `grid-template-columns`. **Fix:** explicit `.dec-row .conf { grid-column: auto !important }` reset. Same pattern needed for F27 + F23 list layouts.

4. **EvalHarness `pct` / `tile` modifier classes gate on TREND, not score.** Canonical contract: `Declined` trend → `.med` modifier (amber). PASS / Improved / Stable → no modifier (green) regardless of score. AC042 at 64% PASS is GREEN. **My initial implementation gated on score (`<90 → med`) which was wrong.** Fixed.

5. **Canned-data extension authority during port iteration.** Found `F26_GUARDRAIL_EVENTS` was missing canonical suffix (`" for RET-244 — output continued without PII."` etc). Extended with `suffix: '...' as const` field — verbatim from canonical HTML. **Pattern for F27 + modals:** if mid-port a canned-data gap surfaces, extend the canned-data verbatim from canonical HTML rather than hardcoding strings in TSX (Rule 17 spirit).

6. **Visual gate iteration count (5 rounds) was high but each round was tight (3-15 min).** Pattern was: Yogesh marks specific drift in screenshot → I check canonical CSS/HTML → fix → re-screenshot → re-submit. Total ~75 min for 5 iterations. Faster than re-port (Day-18 #145 precedent).
