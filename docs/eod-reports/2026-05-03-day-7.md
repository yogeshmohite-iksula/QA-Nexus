## Notable learnings (per kickoff §5 amendment 2026-04-26)

### CI Postgres deficit unmasked by PR #22

Long-standing structural debt previously hidden by `.skip` markers on all DB-touching tests in `apps/e2e`. PR #22 added new tests in `tests/onboarding.spec.ts` (sign-in redirect, magic-link form, /health subsystem readouts, /agents/a1/generate 401) WITHOUT the `.skip` marker — those tests assume the API is up. The API can never be up because PrismaService.onModuleInit fails to connect to a Postgres that doesn't exist in the CI runner. Path δ observability instrumentation (PR #23 commit `4bb478c`) added `if: always()` log dump + ps/memory/disk snapshot, which finally surfaced the api.log Prisma error. Day-8 morning fix tracked as P0 followup `(t)`. **The fact that this debt has existed since CI was set up but only surfaced today is a reminder: `.skip` markers are silent failures-in-waiting. Audit them at every milestone close as a discipline.**

### Three halt-to-root-cause moments today — pattern is becoming project culture

Each saved significant downstream rework. Worth codifying in onboarding:

1. **R2 token scope** (Step 1, AC012 unblock):
   - Symptom: pg_dump → R2 upload returned `AccessDenied`.
   - Speculation bucket would have been: re-create the token, recreate the bucket, debug R2 config exhaustively.
   - Halt-to-root-cause: walked the runbook (`docs/deploy/restore-runbook.md` mention of "SECOND R2 bucket"), recognized that the existing token was scoped only to the first bucket. Yogesh edited the token's bucket scope (no key rotation); 6 min vs hours.

2. **ESLint config collision** (Step 4, Path B → Path D switch):
   - Symptom: 40 `no-explicit-any` warnings in apps/api test files, all `*.spec.ts` paths.
   - Path B (Yogesh's first authorization) would have added 80+ FIXME + eslint-disable annotations across 9 test files + filed a fake-debt followup `(t)` for "M1.5 type-tightening" that nobody should ever discharge.
   - Halt-to-root-cause: noticed all 40 sites were in test files, read the workspace-root ESLint config, found the explicit `'@typescript-eslint/no-explicit-any': 'off'` for test patterns at line 65, then found the apps/api override at line 23-28 with `files: ['**/*.ts']` that silently re-enabled `'warn'` for everything including spec.ts. Path D fix: 6-line config addition to apps/api/eslint.config.mjs scoped to test patterns; 1 file vs 9; aligned with root intent; no debt.

3. **CI flake → CI Postgres deficit** (Step 4 / PR #23 expansion):
   - Initial speculation: bge-large embedding load timeout.
   - Path α probe: re-run E2E to test for flakiness; failed identically (eliminated flake).
   - Path β: changed `EMBEDDING_MODEL_ID` to bge-small; failed identically (falsified bge-large hypothesis — main `68b3ac0` had passed earlier with bge-large).
   - Halt-to-root-cause: instrumented with Path δ (`if: always()` log dump + system snapshot); next failed run revealed `PrismaClientInitializationError: Can't reach database server at localhost:5432`. Real cause is structural (no Postgres service in CI workflow), not env var. Day-8 fix.

**Shared discipline across all three:** SURFACE the discrepancy + ASK before bypassing, prefer evidence over speculation, instrument before iterating, name the falsified hypothesis explicitly so it stays falsified. Today proved each of these saved hours of misdirected work.

### Continue-on-error gotcha

`Run Playwright tests` step in e2e.yml has `continue-on-error: true` ("most specs are .skip during setup" per inline comment — outdated rationale). This masks playwright failures from PR-check rollup AND from `if: failure()` gates on subsequent steps. PR #23 currently shows "playwright SUCCESS" in the GitHub UI despite 22 underlying test failures — meaning a future regression could ship undetected if we relied on that signal. Tracked in followup `(t)` §6 "Bonus fix" — remove `continue-on-error: true` once Postgres deficit is fixed (preserves the safety net of real test signal).
