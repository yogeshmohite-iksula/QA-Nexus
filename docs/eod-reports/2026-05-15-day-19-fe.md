### Lessons-learned — skill v2.2 amendment candidate

**The drift class Yogesh's visual gate caught that the skill diff-probe missed:** 5 canonical sections (People, RCA bars, sd-tabs, group-sep banners, Assignee filter) all existed in canonical HTML but were ABSENT from my Day-18 seed TSX. The skill's v2 diff-probe v2.1.2 uses ARIA-primary structural matching (Hard Rule 18 Day-19 amendment Part 1) which catches MISSING canonical sections at the level the canonical exposes via `role=` and `aria-label=`. But nested `sd-section` siblings inside a single `<aside>` collapse to a single PASS at the parent — the probe can't see whether the parent contains 6 children (canonical) or 4 (Day-18 seed).

**Proposed amendment — `frame-port` skill v2.2 nested-section-count probe:**

1. `extract-spec.mjs` emits a per-parent `nested_section_count` field counting `<div class="sd-section">` (or equivalent semantic) siblings inside each major region.
2. `diff-probe.mjs` walks both canonical + port DOM and verifies the count matches.
3. Mismatch → MISSING SECTIONS RED with specific counts (e.g. "canonical `<aside class='sd-rail'>` has 6 sd-section siblings, port has 4 — missing 2").

Filed for MAIN to consider. Once shipped, future ports won't repeat the round-2/3/4/5 visual-gate iteration that F21 required today.
