## 7. Day-20 systemic lessons archived

Three lessons captured for M4 retro + memory:

1. **Chained-base + squash = guaranteed 1 rebase cycle per chained PR.** When parent PR uses GitHub's "Squash and merge", GitHub deletes the parent branch + auto-closes any PR whose `base_ref` was that branch. The child PR cannot be reopened (`gh pr reopen` fails — base branch gone) nor retargeted (`gh pr edit --base` fails on closed PR). The only path is to open a fresh PR with the same head branch + `base: main`. Hit twice today (#149→#172, #168→#173). **Day-21 prevention:** don't chain PRs unless the parent is going to be merged with "Create a merge commit" (preserves base branch).

2. **Cherry-pick auto-skip in `git rebase` works when parent squash-merged.** `git rebase origin/main` on a chained-PR branch where parent is now in main automatically detects the cherry-pick equivalent and emits `warning: skipped previously applied commit <sha>`. Saves manual conflict resolution on parent's commits.

3. **CHANGELOG conflicts compound across parallel-merging sibling PRs.** All 4 BE PRs (#148, #149, #161, #162) + #168 + #173 touched `docs/CHANGELOG.md` in `[Unreleased]`. Every cascade merge triggered a CHANGELOG conflict on every subsequent PR's rebase. **Day-21 prevention:** filed followup `(dc)` to design a CHANGELOG single-source-of-truth pattern (e.g., per-PR fragment files merged at release time, similar to scriv).

**Bonus lesson:** **prettier doesn't lint git conflict markers** — caught one stale `<<<<<<< HEAD` marker on #148 branch that passed pre-push gates because prettier checked formatting (correct) and CHANGELOG-guard checked entry presence (present). Filed followup `(dd)` for a pre-push hook addition.

**Hard Rule 11 fires Day-20: 0.** Compare Day-19: 4 fires. Discipline-lag effect — the 4 Day-19 cycles trained me to verify GH state with multiple signals before action. Day-20 saved ~30 min by checking `gh api repos/.../pulls/N --jq .head_sha` BEFORE assuming push went through.
