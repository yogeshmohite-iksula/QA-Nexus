### Compound learning (to journal post-merge)

- **2026-05-07 · pnpm shared-package rebuild gate** — When a sibling
  package (`packages/shared`) adds new exports via PR merge to main,
  the FE workspace consumer (`apps/web`) needs `pnpm install` (to
  refresh symlinks) AND `pnpm --filter shared build` (to regenerate
  `dist/` that tsc resolves). Symptom: typecheck reports "no
  exported member 'X'" even though `src/` has the export. Cause:
  shared package emits to `dist/` via `tsc -p tsconfig.build.json`;
  the workspace symlink points at `dist/`, not `src/`.
- **2026-05-07 · F12 RESUME execution** — When a deferred task
  unblocks mid-day, a fast 1-hr execution is feasible if the
  pattern (in this case F13's Pattern B flip) is already in muscle
  memory + the BE schemas are clean + the code-write happens in
  parallel with merge-poll wait. ~165 LOC API client + ~50 LOC
  component change + 5 screenshots in 60 min.
- **2026-05-07 · Cascade-rebase discipline** — When 4 PRs merge in
  rapid succession to main and yours is the slowest CI-wise, expect
  a CHANGELOG/followups conflict. Resolution pattern: keep ALL
  entries chronologically (newest-first journal); never drop an
  entry in conflict resolution.
