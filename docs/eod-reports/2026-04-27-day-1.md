## Key learnings (auto-eligible for /compound-learnings)

- **Worktree-based parallel chats work** — 3 chats × 6+ commits each, merged with only 1 minor MILESTONES.md conflict + 1 baseline-CI hotfix batch. Net: +13 hours of work in ~1.5 hours of wall-clock parallel execution.
- **CI bootstrap chicken-and-egg** — when CI workflow lives on a feature branch, it can't validate that branch's PR (GH uses base-branch workflows). Solution: merge the CI-introducing PR first, accept dirty CI baseline, hotfix the issues that surface, then re-run dependent PRs. Documented as pattern.
- **Dedupe permission rules at consolidation time, not in real-time** — Yogesh's batch-of-31 triage with Cowork pre-classification was 5× faster than the per-prompt accept/reject I'd been doing. Pattern: collect prompts for a session, batch-triage, never approve one at a time.
- **`block-dangerous.sh` regex needs flag-boundary semantics** — bare `--force` substring match was overly broad. `(^|[[:space:]])--force([[:space:]]|$)` is the correct shape. Same pattern would apply to other flag-class regexes if we add more.
- **Settings.json (committed) vs settings.local.json (gitignored)** — Option B (centralize in committed file) makes more sense for a solo-dev / few-contributor project. Skill default Option A (per-machine) is correct for large open-source projects with many contributors who shouldn't share permissions.

These will land in `.claude/memory/general.md` "Compound learnings" section via `/compound-learnings` tomorrow morning.
