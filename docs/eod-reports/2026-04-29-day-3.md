### CI hotfix root-cause trail (filed as followup `(j)`)

Three regressions surfaced when PR #11 opened against the new main `6385e25` (Day-3 evening seed-centralization scaffold):

1. **typecheck + build:** `apps/web/lib/{demo-seed,contexts/*}` started importing `@qa-nexus/shared`, but `pnpm install --frozen-lockfile` does not run workspace `build` scripts and `next build` / `tsc --noEmit` don't trigger the dep build. Fixed at the CI level by adding `pnpm --filter @qa-nexus/shared build` before each dependent job. Tried first to put the build inline in apps/web's package.json scripts (mirroring apps/api), but pnpm 10 scopes `--filter` to the cwd subtree when invoked from a non-`@qa-nexus`-scoped package (`web` is unscoped; `@qa-nexus/api` is scoped — same syntax works there but not in web).
2. **test job — playwright:** `pnpm -r test --passWithNoTests` was rejected by `apps/e2e`'s Playwright CLI (Jest-only flag). Replaced with explicit enumeration: `pnpm --filter @qa-nexus/api --filter @qa-nexus/shared test`. (The `!@qa-nexus/e2e` filter approach failed because the root `qa-nexus` package matches that filter and its own `test` script re-runs `pnpm -r run test` unfiltered — drags e2e back in.)
3. **test job — packages/shared:** `echo "no tests yet" && exit 0` failed when CI appended `--passWithNoTests` (`exit 0 --passWithNoTests` → "too many arguments"). Replaced with `node -e "process.exit(0)"` (Node ignores unrecognized argv). Baked `--passWithNoTests` into apps/api's `jest` invocation; CI no longer passes the flag.

**Detection latency:** commit `6385e25` was pushed direct to `main` (no PR), so CI never validated it. CI runs only on `pull_request` to `main`. PR #11 was the first PR opened against the regressed main and surfaced all three at once.

→ **Followup `(j)` filed:** add `push: branches: [main]` trigger to CI so direct-to-main commits surface regressions immediately. Day-4 morning P1, ~10 min fix.

### Pattern observations from this cascade

- **`gh pr update-branch <N>`** is the right tool for "PR opened before main moved" — it's a one-shot merge of new main into the PR branch, triggers CI re-run, and fails loud on conflicts.
