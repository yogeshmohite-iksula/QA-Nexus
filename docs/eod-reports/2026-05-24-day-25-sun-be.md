## 6. Lessons / patterns logged today

**(a) The verify-first gate works.** Yogesh's `STEP 0 VERIFY` instruction caught a wrong-worktree session start (Claude Code launched in MAIN instead of -backend). Recovery: cross-session `cd /…/-backend` chained on every Bash. Persistent CWD survives within a Bash subshell across turns; `ctx_execute` spawns fresh shells so doesn't persist (caught + corrected this within the cascade).

**(b) GitHub merge-state field semantics.** `merge_commit_sha` is populated on every mergeable PR as the _projected_ merge SHA — it does NOT prove the PR was merged. The authoritative field is `merged: true` + `merged_at: <ISO timestamp>`. Yogesh's "merged" claim 2× preceded the actual merge button click; correct behavior was to hard-stop and ask for re-verification rather than start the rebase on a stale assumption. (Saved at least one fix-forward cycle.)

**(c) Rebase auto-skip in cherry-pick chains.** When rebasing branch B (cut from A's PR head) onto post-A-merge main, git auto-detects A's commits already merged → skips them silently → only B's net-new commits replay. Same-patch detection works even after merge-commit reshape. Worked first try on both #194 and #195's first rebase. Second rebase of #195 (post-#194) did surface a CHANGELOG conflict because both A and B added under `[Unreleased]/Added` — resolved by chronological retention of both blocks.

**(d) NestJS DI scope ≠ module-graph import side-effects.** `useClass`/`useValue` overrides on a provider only apply within the DECLARING module's scope. If an IMPORTED module (e.g., LLMGatewayModule) needs the same provider from its OWN context (via its controller's RolesGuard → AuthService → PrismaService chain), the override doesn't propagate. The fix is NOT to add more overrides at the importer — it's to bypass the import that triggers the cascade entirely. For AC042 harness: import `LLMGatewayService` directly as a provider, skip `LLMGatewayModule` (and thus skip its controller graph).

**(e) Cross-renderer ESM tolerance varies by tool.** Jest's CJS transformer chokes on `better-auth` ESM (the spec test mocks `realtime.gateway` to dodge it). ts-node with `--transpile-only` apparently handles the same chain fine. Worth noting if we ever need to mock the ESM cascade in non-jest contexts.

**(f) Honest blocker surfacing > false-positive results.** Yogesh's "do not silently run a 5-defect AC042" guidance was the right call — running on n=5 with a `REPLACE_ME` key would have produced `AC042 RESULT: top-2 = 0.0% · VERDICT: FAIL` which (a) is statistically meaningless on n=5 and (b) reflects key-misconfig not Sherlock quality. The smoke result is correctly reported as the broken-key noise floor, not as the binding gate. This preserves the gate's signal value for Monday.

**(g) Corpus-expansion plan is the single most valuable artifact tonight.** When the binding gate is blocked on prep work, the highest-leverage move is to make Monday's prep as mechanical as possible. The plan at `.claude/scratch/ac042-corpus-expansion-plan.md` reduces Monday's expansion from "where do I even start" to "step 1: check key, step 2: mine 15 RET tickets per template, step 3: validate schema, step 4: run binding."
