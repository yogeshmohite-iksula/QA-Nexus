### 4-PR auth saga retrospective (#130 → #132 → #137 → #138 → #139)

Five PRs surfaced four root causes — each invisible until the prior was fixed:

1. **`#130` — Zod-v4 dependency requirement** (caught locally Day-17 morning after silent prod crash; root cause: BA 1.6.11 needs `zod@^4.3.6` but root `pnpm.overrides` forced `zod@^3.25.76` across workspace).
2. **`#132` — `allowedAttempts` no-op in BA ≥ 1.6.11** (surfaced via BA's runtime warning after #130's typecheck passed; root cause: GHSA-hc7v-rggr-4hvx hardcoded atomic single-use). #132 removed the dead config line + the regression-pin test.
3. **`#137` — Gmail prefetch consumes the token** (the original symptom — BA's atomic single-use means scanner GET burns the token before user clicks). Intermediate confirm-page pattern: emit a neutral FE page URL; FE renders a "Confirm Sign In" button; token POSTed only on real user click.
4. **`#138` — `@better-auth/core` is a SEPARATE npm package** (Day-17 evening: first auth request after #137 merged crashed `TypeError: z.ipv4 is not a function` at `@better-auth/core/.../ip.mjs:7`). #132's scoped override targeted `better-auth>zod` but not the sibling `@better-auth/core>zod` package. Added the second scoped override; both BA packages now resolve zod@4 inside their `.pnpm` slots while the workspace stays on zod@3.
5. **`#139` — Next.js route group `(auth)` is stripped from URL path** (Day-17 late evening: pilot click rendered Next.js 404 at `/auth/verify-magic-link`; root cause per official docs: parenthesized folders are route groups, NOT URL segments). Dropped `/auth/` from the emitted URL.

**Patterns surfaced (for the Day-17 retro tomorrow):**

- **Pre-push gate boot-smoke is insufficient.** Module-load crashes (`z.coerce.boolean().meta`) AND request-handler crashes (`z.ipv4`) both bypass `tsc + jest + 10s boot`. Followup `(bj)` filed (#132 + #138 + #139 CHANGELOG): need request-level smoke that POSTs `/auth/sign-in/magic-link` with `X-Forwarded-For` against a running server (port-bound, real DB). Captures the entire class.
- **`jest.mock('better-auth')` blinds us to BA-internal crashes.** All 4 auth specs across `apps/api/src/auth/__tests__/` mock the module entry — so the real BA + `@better-auth/core` source never loads during jest. PR #138 added `better-auth-core-ip.spec.ts` that exercises the override via filesystem-resolution checks (transitive `.mjs` subpath import doesn't load through jest's CJS transformer; pinning via `pnpm.overrides` line presence + `.pnpm` slot inspection is the next-best surface).
- **BE↔FE URL contract is a SINGLE source-of-truth problem.** PR #137 emitted `/auth/verify-magic-link` while the FE page mounts at `/verify-magic-link` (route-group strip). Both sides asserted their own URL shape in tests; neither tested the handshake. Followup `(bj)` expansion: gate should fetch the emitted FE URL against a running FE worker.
- **Hard Rule 11 paid off TWICE.** On #138, I caught a mismatch between Yogesh's stated root cause ("BA's getIp throws → uncaught") and BA's actual source ("getIp returns null gracefully + rate-limiter logs warning + skips"). Stopped + asked for the actual Render stack trace. The trace pointed at `@better-auth/core` (a SEPARATE package) — entirely different fix. Would have wasted a deploy cycle implementing the wrong fix.
