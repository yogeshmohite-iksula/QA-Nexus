## Architectural wins worth highlighting (M3 retro candidate)

1. **Env-flag Pattern-A → Pattern-B cutover** (PR #109) — single env var change on Render flips production behavior; zero code touch. Reusable for any future LLM-backed feature.
2. **Schema versioning policy in ADR-013** — additive-only between versions; breaking → `/v2` endpoint. Eliminates the "every PR rewrites the contract" failure mode.
3. **Retry chain observability** — every attempt's outcome lands in DB (`TestCaseGenerationRun.errorReason`) + audit row written BEFORE the 503 throw. Post-mortem of any composer failure is one row read away.
4. **`composer-prompt.ts` as single source of truth** — system prompt + JSON schema + constants in one file with PR-review trigger lock per ADR-013 §"Mitigation plan". FE+1 can import the JSON schema constant directly via shared types.
5. **OFFLINE-mode preservation** — `COMPOSER_OFFLINE=1` env flag OR auto-engaged when `LLMGateway.deferred=true`. Pattern A canned cases still work for dev-without-internet + CI tests don't need real LLM. Day-15 cutover requires zero test changes.
