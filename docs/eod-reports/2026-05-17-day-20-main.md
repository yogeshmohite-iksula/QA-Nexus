**Chained-base + squash gotcha (lesson, captured in memory at `feedback_chained_base_squash_gotcha.md`):** When a chained-base PR's base PR is **squash**-merged (not merge-commit-merged), GitHub auto-closes the child PR because the base branch ref no longer points to a commit reachable from main. The squash collapsed N commits into 1 + retargeted to main, so the child's base-branch-ref dangles. **Workaround:** child re-opens as new PR retargeted to main; content unchanged, just a fresh PR number. **Fix-forward for future cascades:** either (a) merge-commit instead of squash for chained-base parents, or (b) FE/BE rebases the child onto main pre-merge of parent. Documented in PR #166 cascade brief for future Day-N close ceremonies. 6. #169 F20 React port (FE+1 promote, CHANGELOG rebase against #161/#162/#168, MERGEABLE within ~5 min)
