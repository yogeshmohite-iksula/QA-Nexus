## Lessons learned (Day-24)

1. **Scaffolds-in-`.claude/scratch/` paid off again (3rd validation).** The 108-line migration scaffold + 127-line controller scaffold + 119-line cron scaffold from Day-23 buffer compressed the P0 build from estimated 4hr to ~2hr. Code from scratch would have meant rethinking the schema CHECK constraint enum values, time_range_key normalization scheme, and cron timing rationale during the build. With scaffolds: just translate scaffold → real file + extend. ~50% time savings. Promote pattern to Day-25 candidate: pre-stage Day-26 work during Day-25 buffer if AC042 gate passes early.

2. **prettier --write BEFORE git add eliminates husky retry cycle (Day-23 lesson applied 2×).** Day-24 P0 push: pre-staged prettier write across changed paths → push passed all 4 gates first try. Day-24 P1 push: same pattern → first-try success. Zero husky retry cycles today vs Day-23's one retry. Promote to Day-26 EOD recipe: `pnpm exec prettier --write <changed-paths>` between heredoc-write and `git add`.

3. **AuthService mock at jest.mock module boundary works for ANY controller spec.** The pattern `jest.mock('../../auth/auth.service', () => ({ AuthService: class FakeAuthService { resolveSession = jest.fn(); } }))` from Day-21/22 ESM-transitive-import workaround is reusable across all future controller specs. Day-24 ReportsController spec applied it cleanly. Add to `.claude/memory/feedback_jest_better_auth_mock_pattern.md` candidate.

4. **commitlint header-max-length=100 is tighter than the average M5 tag conventions.** "[m5 day-24 — completes ADR-020 wire-up]" pushed header to 116 chars on first commit. Shortened to "[m5 day-24]" cleanly. Lesson: tag suffixes in commit headers should stay under 15 chars beyond the type+scope+summary; PR title can carry the verbose tag without commitlint check.

5. **RBAC RolesGuard transitively requires AuditService in test modules.** Day-21/22 spec patterns missed this. Day-24 ReportsController spec fixed via explicit `{ provide: AuditService, useValue: { writeNonBlocking: jest.fn(), write: jest.fn() } }` provider. Add to memory candidate for future controller-spec authoring.

6. **PR #189 lingering open status was a productivity win, not a blocker.** Yogesh said "#189 is MERGED" in the morning trigger but the actual state was OPEN. Made the reasonable call: branch Day-24 work from #189 head (`f1f2b30`) rather than waiting. Both Day-24 PRs include #189's commits transitively. If #189 evolves through review, both Day-24 branches rebase cleanly. Doctrine validated: branch-from-pending-PR-head + document dependency in commit msg + PR body works for short-lived parallel work.
