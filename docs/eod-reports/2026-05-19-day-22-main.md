## 7. Lessons learned (Day-22 retro candidates)

Three retro candidates filed today as memory entries; one-line summary each + link to the full memory file. Surface together at M5 retro under umbrella theme **"multi-agent coordination friction"**.

### 7.1 Session resume-state pulse (`feedback_session_resume_state_verification.md`)

**Pattern:** Every session resume MUST start with a 4-invariant pulse (current branch · origin/main HEAD · open PRs · working-tree status) via one `ctx_batch_execute` call. ~5 sec wall-time; prevents the 10-min branch-drift recovery cycle that bit Day-22 multiple times.

**Cost-benefit:** 20 sec/day pulse cost vs ~40 min/day branch-drift recovery cost = 120× ROI.

**Validated TWICE in the same session it was filed:** (1) on session resume Day-22 PM — surfaced PR #186 + channel-naming deviation + 3 memory files already on `docs/main-day-22-eod` (no re-commit); (2) on the PR #187 -frontend worktree task — caught that the branch was already pre-staged by Yogesh's earlier attempt and skipped the redundant `reset --hard` (5-min ETA vs 10-15 min estimate, a ~50% efficiency win on a single task).

### 7.2 Branch-lineage drama (`feedback_branch_lineage_drama.md`)

**Pattern:** Multi-agent sessions share one working tree. Background BE+1/FE+1 agents do `git checkout` + `git stash` + `git commit` independently of MAIN. Result: 4 incident classes (silent branch reset · wrong-branch commit hijack · stash drift across branches · uncommitted-WIP leakage blocking pre-push typecheck).

**Workaround that proved out:** `cat > /tmp/file && git checkout -B <branch> origin/main && cp /tmp/file <path> && git add + commit` ("cat-heredoc workaround"). Day-22 cost: ~30 min recovery across 4 incidents = ~8% of compressed 6.5hr session.

**Invariant proposed for Day-23+:** every `git checkout -b` MUST source from `origin/main` (after `git fetch`), NEVER from local HEAD. Enforceable via pre-tool-use Bash hook that asserts `git rev-parse HEAD == <expected SHA>` before any Edit/Write commits.

**M5 retro candidate** — surface as a multi-agent-coord issue, NOT personal discipline.

### 7.3 Parallel-chat routing paste drift (`feedback_parallel_chat_routing_paste_drift.md`)

**Pattern:** Single human × 3 windows (MAIN + BE+1 + FE+1) + ~150 pastes/day → ~5-7% paste error rate → **10 misroutes observed Day-22** (vs 7 Day-21). Destination labels (`╔══ PASTE TARGET: X ══╗`) work for the next paste but can't 100% prevent muscle-memory paste into the wrong window during Cmd-Tab + Cmd-V sequences.

**Cost:** ~4 min per misroute (detection + redirect + re-paste + lost focus) = ~40 min Day-22.

**Mitigations (cheap → expensive):** window-color-coded titles (cheapest, OS-level) · paste-target validation in chat session (medium) · paste-pin per window (most ergonomic, requires hook plumbing) · **NEW 4th mitigation discovered Day-22 PM: park-at-destination-scratch when destination chat is offline/unreachable** (filesystem-park beats paste-routing entirely — the FE+1 Day-23 kickoff brief was parked at `-frontend/.claude/scratch/` so Yogesh `cat`s it tomorrow AM at FE+1 session restart; routes around the misroute pattern by removing the paste from the loop).

**Recommended Day-23 experiment:** window-color-coding via Claude Code CLI session config OR OS-level window-rename script. The park-at-scratch pattern is now an everyday tool for offline-destination handoffs.
