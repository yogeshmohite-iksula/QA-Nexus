## §7 — Lessons learned (M5 retro candidates)

1. **A schema-bridge failure presents as a zero-hypothesis DEGRADED state, not a
   calibration miss.** top-2 0% + n/a calibration ⇒ suspect the parse/validation layer
   (agents returning `[]`), not the model's reasoning.
2. **Smoke 1 case before the binding eval.** 4 Groq calls localized the bug that a
   200-call / 60-min full run only flagged as a vague 0%. Cheap structural check first,
   expensive authoritative gate second.
3. **Plan A2 calibration-nudge is a generalizable LLM-overconfidence remedy** (0.57 →
   1.00, no code change). Reusable for any LLM feature emitting a confidence (Composer,
   Curator) → promote to PM1_PATTERNS.md.
4. **AC042 corpus provenance is LLM-assisted** (Codex labels + human spot-check) — an
   ADR-022 §5.9 reserve precedent; the gate measures LLM-vs-LLM agreement, not pure-human
   ground truth. M6 corpus-governance item.
5. _(process)_ During a **rebase**, `--theirs` is the branch being replayed (sides swap
   vs. a merge) — use `git checkout <ref> --` to take a specific side's version
   unambiguously.
