## §9 Day-25 Sun additional lessons (late-night batch — append to §"lessons")

5. **Flat `rightAlignFrom` rule for tables is wrong** when adjacent columns have different content types (numeric right + pill left). Use explicit per-column `align[]` array — header AND value must use the same alignment to avoid drift.

6. **`table-layout: fixed` + `width: ${100/N}%` = equal-distribution columns.** Pairs well with `whitespace-nowrap` on cells to prevent line wrap. Pattern is the "trust the user" version of column sizing — give every kind equal weight even if content varies.

7. **Tailwind class-conflict order matters.** Later class wins; `text-right` passed via Td `className` overrides the helper's `text-left` default. Lesson: when designing reusable cell components, document which utility classes are settable via prop and which are fixed defaults — or use TW merge utility to normalize.

8. **Visual-gate feedback "column spacing" can mean three different things**: (a) gap between cell content too tight/wide, (b) header position different from value position, (c) columns unevenly distributed. Each needs a different fix. When unclear, ASK before guessing — same lesson as F19 R5 env-chip flip-flop.

9. **`table-layout: fixed` requires whitespace-nowrap on long text cells** (e.g. "RET-3354 · Refund webhook timeout fix") OR explicit truncation, otherwise text wraps to multiple lines within the fixed column width. Used nowrap throughout F23 tables — long text will horizontal-scroll if needed (parent `overflow-x-auto`).

10. **End-to-end re-port in one commit beats incremental patching** when canonical-vs-port drift crosses many regions. The `ce49b25` commit lands all 9 regions (Configure, KPI, 6 charts, result-head, 6 tables, modal, AI tile, saved cards, scheduled rail) in one atomic change — far cleaner git history than 10+ tiny commits with overlapping fixes.
