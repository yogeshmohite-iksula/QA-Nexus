## What worked (5)

1. **3-agent parallel-chat orchestration via paste-blocks.** Day-15's 15-PR record relied on MAIN coordinating BE+1 + FE+1 chains that executed independently and synced only at merge boundaries. No automation needed at this scale; the paste-block protocol cleared every coordination beat. Same pattern carried M2 from 4-day plan → 2-day actual.

2. **Compression cadence as muscle memory.** M2 = 2× (4-day → 2-day). M3 = 1× (5-day plan → 6-day actual, but 3.6 working days = ~1.4× efficiency vs original 4-week M3 baseline in M0 plan). M4's 4-day target is only feasible because the team trusts the cadence now.

3. **Hardcoded-fallback pattern.** When Cloudflare Pages env-var injection silently failed (M3 drift #2), `#122` hardcoded production baseURL fallback unblocked production without waiting for upstream Cloudflare fix. Pattern re-used immediately in `#129` for absolute callbackURL. Lesson: when a third-party config-injection path proves unreliable, fall back to source-controlled value with explicit env-override — don't pretend the broken path will heal.

4. **Path C zero-code LLM cutover (#115 LLMGateway DB-source bridge).** Composer + Curator shipped before F26 admin UI (deferred to M5) by reading from `llm_providers` table at boot. Single seed script flipped DEFERRED → real in production with zero FE/BE code changes post-merge. Lesson: prefer config-bridge cutover over UI dependency for risky upgrades.

5. **Cascade-rebase runbook (`(ar)` followup).** Day-13 GHA outage left 9 PRs cascade-conflicted on resume. The `git rebase --onto origin/main <upstream-squashed-commit>` pattern was discovered Day-13, formalised Day-14, and re-applied Day-15+. By Day-17 every cascade-storm cleared within one polling cycle.

## What hurt (5)

1. **9-PR BetterAuth fix chain Day-15 → Day-17.** Day-15: 5 PRs (#119 BE mount widen → #120 FE basePath → #122 Cloudflare env-var hardcoded fallback → #123 trustedOrigins → #124 CORS preflight). Day-17 morning: #129 absolute callbackURL. Day-17 evening: 4 more (#137 Gmail prefetch intermediate confirm page, #138 zod scoped override for `@better-auth/core` family member + 4-layer proof, #139 drop `/auth/` prefix in sendMagicLink URL, plus BetterAuth ≥1.6.11 hardcoded single-use atomic consumption discovery — `allowedAttempts` no-op). Nine PRs for what should have been one. Each layer revealed the next; all caught user-side, not in pre-deploy smoke. Single biggest pain of the milestone. **Action:** pre-push request-level smoke gate (followup `(bo)`).

2. **Render Free SMTP block discovery (Day-16).** Render Free blocked outbound SMTP since Sept 2025 (policy change post-ADR-008 deploy-target verification). Auth chain landed clean Day-15 but emails didn't deliver. ADR-008 → ADR-018 migration thrashed a full day of close-ceremony momentum. Lesson: deploy-target verification gate at every BE ADR (catches this whole class).

3. **Cross-origin `callbackURL` bug (#129).** MAIN brief's earlier sample assumed `signIn.magicLink({callbackURL: '/home'})` would resolve to FE origin. It resolves to API origin (BetterAuth client basePath). I (MAIN) drafted the brief without reading BetterAuth's request-context behaviour first. Caught Day-17 only.

4. **gitleaks fixture-key false positive blocked 4 PRs simultaneously (Day-15).** `F16b-PB-generate` test fixture string entropy 3.58 = generic-api-key match. Single allowlist fix (#121) unblocked #117/#118/#119/#120 in one cycle, but the simultaneous blockage cost ~30 min of triage. Lesson: gitleaks allowlist regex for `'F\d+[a-z]?-\w+-\w+'` pattern (followup `(bd)`).

5. **Quota / API rate limits as recurring blocker.** Day-13 evening Anthropic API rate limited the team for ~30 min. Day-16 Yogesh's MAIN session hit personal quota mid-cross-FE-smoke. Day-17 was entirely a refresh day. ~1.5 working days lost to quota across M3. No clean mitigation — just a fact of free-tier usage at this volume.

## What to drop (3)

1. **Orchestrator drafting briefs without reading the target files.** Day-17 BE+1 + FE+1 stops are evidence: I (MAIN) wrote sequences referencing schema fields or HTML lines I hadn't verified. Six BE schema-mismatch catches + 3 FE v2-HTML-mismatch catches today alone. Action: Phase 0 pre-read every M4 brief.

2. **Assuming package versions support documented features.** BetterAuth 1.2 ≠ 1.5+ `allowedAttempts` — should have grep'd `package.json` first before referencing options. Same with wrangler@latest requiring Node 22 (caught only via deploy failure, not via package.json read). Action: pin all installed pkg versions in brief context before suggesting API changes.

3. **Bundling unrelated work in fix PRs.** Day-12 #79 prettier cascade pattern (one PR fixing prettier across 4 files because 4 owners had each tripped it independently) was efficient at that scale but obscured ownership. Day-15 #121 fixed the same pattern more cleanly (1 allowlist entry, 1 owner). Action: name the bundled scope explicitly in PR title (e.g., `(unblocks #117 #118 #119 #120)`).
