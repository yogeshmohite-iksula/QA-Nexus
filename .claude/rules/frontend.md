# Frontend rules (apply to apps/web/\*\*)

These rules govern every change inside `apps/web/`. Authority order per CLAUDE.md: **PM1_PRD > PM1_ERD > M0_v8 > 01_SYSTEM > tech-project-forge > MCP > library defaults**.

## Design tokens

- Use **only** tokens from the `@theme` block in `apps/web/app/globals.css`.
- **No Material Design 3 tokens.** The `enforce-design-tokens.sh` PreToolUse hook blocks: `primary-container`, `on-primary`, `surface-tint`, `surface-bright`, `surface-container-*`, `tertiary-*`, `inverse-*`, `surface-variant`.
- **Teal `#2DD4BF`** for **system** actions ONLY — Sign In, Save, Generate, Approve, primary CTAs.
- **Violet `#A78BFA`** for **AI** surfaces ONLY — agent labels, confidence lanes, AI badges, agent run cards.
- Hex colors outside the locked whitelist (`enforce-design-tokens.sh` line 31) are blocked. Use CSS vars or whitelisted arbitrary values.

## Components

- **shadcn/ui only** for primitives: `Button`, `Input`, `Dialog`, `Card`, `Toast` (Sonner), etc.
- Custom components live in `apps/web/components/{auth,workspace,agents,defects,reports,nav}/`.
- Icons: `lucide-react`. Forms: `react-hook-form` + `@hookform/resolvers/zod`. Animation: `framer-motion`.

## Responsive (CLAUDE.md Rule 12)

- **Mobile-first.** Base styles target ~320 px (iPhone SE). Progressively enhance via Tailwind breakpoints `sm 640 / md 768 / lg 1024 / xl 1280 / 2xl 1536`.
- **No fixed pixel widths on layout containers.** Enforced by `enforce-rwd.sh` hook (MS0-T034) — blocks `w-[≥200px]`, `max-w-[1600px]`, `h-[≥200px]`. Use `w-full`, `max-w-{md|lg|xl|2xl}`, `flex-1`, grid.
- **Component max-widths** are fine when semantically correct: forms ≤ 480 px, reading content ≤ 768 px (`max-w-[480px]`, `max-w-[768px]` are explicitly allowlisted by the hook).
- **Tap targets ≥ 44 × 44 px** (WCAG 2.5.5) on every interactive element.
- **No horizontal scroll** at any viewport ≥ 320 px wide. Test at 320 / 768 / 1024 / 1440 / 1920 minimum before commit.
- **Modals** (Stage 1120×860, Edit 960×720, Picker 720×640, Confirm 480×360 per `01_SYSTEM.md`) → **full-screen Drawer sheets on mobile**, declared sizes on desktop.
- **Visual confirmation gate** (CLAUDE.md Rule 13): for every newly developed/refactored screen, run `/ui-check <route>` and wait for explicit "looks good — commit?" approval BEFORE running `git commit`.

## State + data

- **TanStack Query v5** for ALL API calls. Never use raw `fetch` in components — wrap in `useQuery` / `useMutation` hooks under `apps/web/hooks/`.
- **Zod schemas from `packages/shared`** for ALL form validation AND ALL API response parsing. Frontend imports the same schemas the backend declares (single source of truth).
- **react-hook-form + `zodResolver`** for ALL forms. No ad-hoc `useState` form state.
- Server state: TanStack Query. Client state: `useState` / `useReducer` / Zustand if needed (no Redux).

## Forbidden imports

The `enforce-pm1-stack.sh` hook blocks these in `package.json`. Do not attempt to import:

- `@mui/*`, `mui` — Material UI
- `@chakra-ui/*` — Chakra UI
- `@mantine/*` — Mantine
- `daisyui`, `material-tailwind`
- `react-bootstrap`
- `antd` (Ant Design)
- `@redis/*`, `redis`, `ioredis`, `bullmq` (these are FE-blocked too, even though the use case is BE)

If you genuinely need a primitive shadcn/ui doesn't have, ask Yogesh first — never reach for a forbidden library.

## File layout convention

```
apps/web/
├── app/                    # Next.js App Router pages (file = route)
│   ├── (auth)/             # route group — auth screens
│   ├── (workspace)/        # route group — workspace screens
│   └── globals.css         # @theme tokens (single source for design tokens)
├── components/
│   ├── auth/               # F06, F06b, F07, etc.
│   ├── workspace/          # F08, F08b, F09, F10
│   ├── agents/             # F18-F23 (violet AI surfaces)
│   ├── defects/            # F24-F26
│   ├── reports/            # F27
│   ├── nav/                # nav primitives, breadcrumb, header
│   └── ui/                 # shadcn primitives (auto-generated by `pnpm dlx shadcn add`)
├── hooks/                  # TanStack Query + custom React hooks
├── lib/                    # framework-agnostic utils (cn, env, fetcher)
└── app/api/                # Next.js route handlers — DEPRECATED for PM1; all server logic lives in apps/api
```

## Frame-port protocol (per `feedback_frame_port_protocol.md`)

When porting one of the 41 locked HTML frames in `QA Nexus/PM1/PM1_UI_v2/`:

1. Read the locked HTML frame — never modify it.
2. Translate to React component(s) under `apps/web/components/<area>/`.
3. Reference the source: `// Implements F06 Sign In · see PM1_UI_v2/frame  html view/F06 Sign In.html`.
4. Run `pnpm build`, `pnpm lint`, husky pre-commit hooks.
5. **Resize-test** at 1600×800 via Playwright MCP before declaring done.
6. Save screenshot to `docs/screenshots/`.
7. Run `/ui-check <route>` for the Rule 13 visual-confirmation gate.
8. Only commit after Yogesh's explicit "looks good — commit" approval.
9. Document any deviation from the locked frame inline (in code comments).
