You asked: "How do we test 3DS challenge flows?"

Test 3DS by validating the challenge invocation (issuer ACS redirect), webhook delivery (`payment_intent.requires_action` and `succeeded`), and fallback handling when the challenge times out or the user fails OTP. Use Stripe's test card 4000 0027 6000 3184 to force a challenge in staging. For Razorpay, our team's gone with frictionless-first then forced-3DS rerun (see API Testing Best Practices §4).
